# InsightGraph 演示版成功门槛

本文档定义每种报告强度的最小成功标准。当任务质量不达标时，系统会明确报告失败原因。

## 通用门槛

所有强度级别共享以下最低要求：

| 指标 | 阈值 | 说明 |
|------|------|------|
| evidence_count | >= 8 | 收集到的总证据条数 |
| verified_evidence_count | >= 8 | 已成功 fetch 验证的证据条数 |
| successful_llm_call_count | >= 2 | 成功的 LLM 调用次数 |
| report_chars | >= 2000 | 报告正文长度（字符数） |

## 按强度分级门槛

### concise（精简版）

| 指标 | 要求 |
|------|------|
| 目标字数 | 800 |
| 搜索限制 | 6 次 |
| 最大工具调用 | 24 次 |
| 最小已验证证据 | 5 条 |
| 最大 token | 100,000 |
| **适用场景** | 快速验证、概念演示 |

### standard（标准版）

| 指标 | 要求 |
|------|------|
| 目标字数 | 7,000 |
| 搜索限制 | 30 次 |
| 最大工具调用 | 320 次 |
| 最小已验证证据 | 18 条 |
| 最大 token | 500,000 |
| **适用场景** | 一般业务研究、演示 |

### deep（高强度版）

| 指标 | 要求 |
|------|------|
| 目标字数 | 8,000 |
| 搜索限制 | 55 次 |
| 最大工具调用 | 700 次 |
| 最小已验证证据 | 24 条 |
| 最大 token | 2,000,000 |
| **适用场景** | 深度公司分析、竞争研究 |

### deep-plus（极限高强度版）

| 指标 | 要求 |
|------|------|
| 目标字数 | 18,000 |
| 搜索限制 | 180 次 |
| 最大工具调用 | 4,000 次 |
| 最小已验证证据 | 72 条 |
| 最大 token | 40,000,000 |
| **适用场景** | 全面行业报告、学术级研究 |

## Critic 质量门控

除上述门槛外，Critic 阶段还需满足（严格模式下）：

1. **全部引用通过**：所有 citation 的 support_status 必须为 "supported"
2. **Section 覆盖充分**：每个必需 section 有足够的已验证证据
3. **实体覆盖充分**：查询涉及的所有实体均有足够证据
4. **主题覆盖率**：report 覆盖研究中识别到的至少 80% 主题
5. **无弱结论**：所有结论均能映射到已验证证据

## 报告质量评分

报告整体评分 >= 80 分且无空 section 视为通过：

| 维度 | 权重 |
|------|------|
| 引用支撑 | 25% |
| Section 覆盖 | 20% |
| 来源多样性 | 20% |
| 证据深度 | 15% |
| 报告深度 | 10% |
| 风险覆盖 | 5% |
| 事实映射 | 5% |

## 失败处理

- Critic 失败时自动触发 replan + recellect，最多重试 3 次
- Reporter 后的质量门控失败时，任务标记为 failed 并展示具体不达标项
- Failed 任务可通过 Dashboard 或 API 重试
