# InsightGraph v0.2.0 稳定服务器演示版执行计划

> 目标：把当前版本整改到可稳定上云、可演示、可排障、可恢复。

## 1. 当前结论

当前版本已具备核心研究链路、API、Dashboard、异步任务、SQLite job 后端、LLM 路由、搜索多选、报告强度档位、质量诊断、部署文档、Docker 基础文件、演示检查脚本。

当前阻塞点：

- `ruff check .` 会失败
- `check_search_provider.py` / `check_llm_provider.py` 在 Windows 读取 `.env` 时存在编码风险
- `validate_demo_env.py` 逻辑和文档不一致
- `Dockerfile` 安装顺序有风险
- `compose.yaml` 默认启动 Nginx，首跑容易被证书/端口挡住
- CI 没有 Docker 构建和 compose 验证

## 5. 每步验收

- `python -m ruff check .`
- `python -m pytest -o addopts=`
- `python scripts/validate_demo_env.py`
- `python scripts/check_search_provider.py --provider mock`
- `docker compose config`
- `docker build -t insightgraph:review-v0.2.0 .`
- `docker compose up -d`
- `/health` 返回 200
- `/dashboard` 可打开

## 6. 操作 Runbook

### 6.1 本地检查

```bash
python -m ruff check .
python -m pytest -o addopts=
python scripts/validate_demo_env.py
python scripts/check_search_provider.py --provider mock
```

### 6.2 容器检查

```bash
docker compose config
docker build -t insightgraph:review-v0.2.0 .
docker compose up -d
curl http://localhost:8000/health
```

### 6.3 故障排查

- `ruff` 失败：先修脚本长行和 unused import
- `.env` 失败：确认按 `utf-8` 读取
- `docker build` 失败：先看 Dockerfile 是否在 install 前已复制源码
- `compose` 失败：先确认 Nginx 是否改成可选 profile
- `dashboard` 失败：先看 `/health` 和容器日志
