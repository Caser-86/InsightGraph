# Demo Pack Design

## Goal

Create a small, reproducible MVP demo package that shows InsightGraph producing a high-quality technical-review report for AI coding agents.

## Audience

The primary audience is a technical reviewer evaluating whether InsightGraph can produce evidence-grounded competitive analysis with visible workflow and LLM observability.

## Scope

- Add a generated showcase report under `reports/`.
- Add a demo guide under `docs/` with offline and live-LLM commands.
- Add a short README entry that points to the demo guide and showcase report.
- Keep the default path offline/deterministic and make live LLM usage explicit.

## Report Topic

The showcase report compares Cursor, OpenCode, Claude Code, GitHub Copilot, and Codeium/Windsurf from a technical evaluation perspective.

The report should cover:
- Executive summary.
- Product and architecture positioning.
- Competitive matrix.
- Developer experience and integration model.
- Enterprise adoption risks.
- Technical-team recommendation.
- References generated by the workflow.

## Demo Flow

The demo guide will provide three paths:

1. Offline smoke demo using the deterministic default workflow.
2. Live LLM demo using `--preset live-llm` and current OpenAI-compatible environment variables.
3. Observability demo using `--show-llm-log` or `scripts/run_with_llm_log.py` to show safe metadata without prompts or completions.

## Output Files

- `reports/ai-coding-agents-technical-review.md`
- `docs/demo.md`
- `README.md` update in the documentation entry and quick-start section.

## Validation

- Generate the showcase report through the project CLI or script rather than writing it from scratch.
- Run source validation on the generated report.
- Run `ruff check .` and `pytest` if production code changes; for docs/report-only changes, run whitespace checks and relevant script validation.

## Non-Goals

- No API auth implementation.
- No frontend implementation.
- No checkpoint/resume implementation.
- No change to default offline behavior.
- No prompt, completion, raw response, or API key logging.

## Open Decisions

The generated live report quality depends on the configured LLM provider and live/search environment. If the live report is weak or fails, fall back to the offline output plus a small editorial polish pass that preserves citations and does not invent facts.
